# Contributing to urmode

Thank you for your interest in contributing to urmode.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature or bugfix
4. Make your changes
5. Test your changes thoroughly
6. Submit a pull request

## Development Setup

### Prerequisites

* Python 3.7 or higher
* Windows 10 (1809+) or Windows 11

### Installation

```bash
git clone https://github.com/Mizokuiam/urmode.git
cd urmode
pip install -r requirements.txt
```

### Running from Source

```bash
python urmode.py
```

### Building

```bash
python build.py
```

## Code Style

* Follow PEP 8 guidelines
* Use meaningful variable and function names
* Add comments for complex logic
* Keep functions focused and single-purpose

## Testing

Before submitting a pull request:

1. Test the application on a clean Windows installation
2. Verify all menu options work correctly
3. Test startup functionality
4. Check auto-switch feature with different timezones
5. Ensure no errors in urmode_error.log

## Pull Request Process

1. Update README.md if you add new features
2. Describe your changes clearly in the PR description
3. Reference any related issues
4. Ensure your code builds without errors
5. Wait for review and address any feedback

## Reporting Bugs

When reporting bugs, please include:

* Windows version
* Steps to reproduce
* Expected behavior
* Actual behavior
* Contents of urmode_error.log (if applicable)

## Feature Requests

Feature requests are welcome. Please:

* Check if the feature already exists
* Explain the use case clearly
* Describe the expected behavior

## Questions

If you have questions, please open an issue with the "question" label.

## Code of Conduct

* Be respectful and inclusive
* Provide constructive feedback
* Focus on the code, not the person
* Help others learn and grow
